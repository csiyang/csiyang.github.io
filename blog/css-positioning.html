Technical Blog: CSS Positioning

Relative vs Absolute vs Fixed

The position property is one of the many CSS elements that help with the ‘positioning’ of different elements of a website. While its importance may seem trivial to some at first, it is one of the most important properties to give your site the best possible user experience.

There are 5 total properties for position, which are Static, Relative, Absolute, Fixed and Sticky. The Static is the default layout for a web page, based on the element above, it will place the next element below. (For a quick example of Sticky, have a look at the nav bar as you scroll!)

Relative

The Relative property builds upon Static by allowing the programmer to adjust where the element would have been had it been static. The start location is based of the element that is before it.

Absolute

This property in some ways is very similar to relative, but where it differs is it removes the element from the ‘flow’ of the page. To put simply, instead of elements going one after the other, it allows the element to stack together.

Fixed

Much as the name describes, this property will make the element relative to the user’s screen and no matter how much the page is scrolled the element will always remain there.
